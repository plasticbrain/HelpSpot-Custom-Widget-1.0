HelpSpot Custom Widget Plugin 1.0

HelpSpot is an excellent, web-based Help Desk/Support Center solution. While they are constantly working to improve their software, one of the things they have not gotten around to updating yet is their "widgets" (the contact forms that you add to your page, which integrate into your HelpSpot account).

That's where this plugin, HelpSpot-Custom-Widget, comes in. It basically clones the default HelpSpot widget, but gives you much more control over the forms, as well as many more customization options.

Live Demo

To see a live demo please visit http://labs.mikeeverhart.net/helpspot.

What's New in This Version?

This is the first "official" (non-beta) release of this plugin, so there were several improvements made:

  • The most notable improvement was that the forms are now loaded inside of an AJAX "ligthbox" instead of an iFrame. Let's face it... iFrames are so 1999. Plus, by loading the forms in a lightbox, things just work better: The forms automatically resize better, we no longer have to include our stylesheets and javascript libraries multiple times, and more!

    One of the only downsides -- if you want to call it that -- of using the lightbox is that everything now depends on AJAX -- loading the forms, submitting the data, etc. However, on a positive note, this really improved the "flow" of things (improving user experience). This also gave me the opportunity to implement the jQuery AJAX Validation library to help you easily validate your form fields! I also think that, due to the improvements in user experience, it was a fair trade-off for any advantages that may have been lost by no longer using an iFrame.

  • The ability to use and create themes; making it even easier to customize how your forms look. There are four [very basic] themes, not counting the default theme, included.
  • Even more customization options!

Requirements

There are only two things required to use this plugin:

  • Obviously, you need an account with HelpSpot
  • Secondly, this plugin requires jQuery to work. If you're already using jQuery on your site, great! If not, you will need to include it for the plugin to work (there are instructions below on how to do this.)

Installing and Using the Plugin

To install the plugin simply upload the entire directory (HelpSpot-Custom-Widget/) to your server. Generally, it's recommended that you upload it to the root directory of your server, although this is not necessary.

Initializing the Plugin

You must include (at a minimum) the following code on any page where you want the Widget to display. NOTE: You may need to adjust the file locations depending on where you uploaded the plugin files!

			
			<!-- include our base stylesheet -->
			<link href="HelpSpot-Custom-Widget/widget.css" rel="stylesheet" type="text/css" media="screen" />
			
			<!-- include jQuery -->
			<script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.6.2/jquery.min.js></script>
			
			<!-- include widget.js -->
			<script type="text/javascript" src="HelpSpot-Custom-Widget/widget.js"></script>
			
			<!-- setup and initialize the widget plugin -->
			<script type="text/javascript">
			$(function(){
			
			   // you could probably bind this to just about any element...
			   $('html').HelpSpotWidget({
				
			       // the base url to where HelpSpot is installed (***include trailing slash***)
			       host: 'http://support.yourdomain.com/',
				
			       // the base url to where the widget plugin is located (***include trailing slash***)
			       base: 'HelpSpot-Custom-Widget/'
			   });
			});	
			</script>

Customizing the Plugin

A Brief Introduction to Customizing the Plugin

You have the ability to customize many aspects of both the tab and the form. You can view the Customization Options and Text Labels sections in the table above to see what all you can customize. Here is a vey basic example of how you can customize the plugin:

			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       
			       // Customize the Tab
			       //-----------------------------
			       
			           // The type of tab to show (support, feedback, help, questions) 
			           tab_type: 'feedback', 
			           
			           // The color of the tab (accepts hex codes and plain-text)
			           tab_color: '#c00c00', 
			           
			           // Where to place the tab on the page (left, right)
			           tab_alignment: 'right', 
			           
			           // How far from the top of the page to display the tab. (accepts px and %)
			           tab_top: '40%', 
			       
			       // Customize Text Labels
			       //-----------------------------
			       
			           // the text to use on the form submit button
			           submit_button_text: 'Submit', 
			           
			           // The headline that the user sees upon success
			           success_headline_text: 'Your Message Has Been Received!', 
			           
			           // a small blurb of text the user sees after they submit a ticket
			           success_body_text: 'We will review your submission and someone will reply to you soon.' 
			   });
			   
			});	
			</script>

Pre-populating Form Fields with a User's Data

You also have the ability to pre-populate the value of the form fields. This allows you to customize the form per user. See the Pre-Populating Fields section in the table above to see what all you can do. Here is a basic example of how to pre-populate the form fields:

			<?php
			// Presumably, this data would be loaded dynamically from a database or other source 
			// For our demo, we'll just create our own array
			 $user = array(
			    'first_name' => 'John',
			    'last_name' => 'Doe',
			    'email' => 'john@doe.com',
			    'phone' => '555-123-4567'	  
			);
			?>	
			
			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       default_first_name: '<?php echo $user['first_name']; ?>',
			       default_last_name: '<?php echo $user['last_name']; ?>',
			       default_email: '<?php echo $user['email']; ?>',
			       default_phone: '<?php echo $user['phone']; ?>'
			   });
			   
			});	
			</script>

Using Themes

New to this version of the plugin is the ability to use Themes to easily change the way your forms look. Using a theme is pretty simple -- It only requires you to add one more line of code that will include the stylesheet for the theme you want to use.

How to Use Themes

To use a theme, simply add the code found below (line #5) anywhere AFTER you've included the widget.css base stylesheet (Seen on Line #2):

			<!-- include our base stylesheet -->
			<link href="HelpSpot-Custom-Widget/widget.css" rel="stylesheet" type="text/css" media="screen" />
			
			<!-- include the stylesheet for our theme -->
			<link href="HelpSpot-Custom-Widget/themes/{THEME-NAME}/style.css" rel="stylesheet" type="text/css" media="screen" />

Of course, you would replace {THEME-NAME} with the name of the theme you want to use. Available themes can be found in the themes/ directory.

Say, for example, that you wanted to use the included clean theme. This is the code you would use:

			<!-- include the stylesheet for our theme -->
			<link href="HelpSpot-Custom-Widget/themes/clean/style.css" rel="stylesheet" type="text/css" media="screen" />

Creating Your Own Themes

It's not hard to create a new theme, but it does require you to have some knowledge of CSS. Follow these steps to create your own theme:

  • Make a copy of the theme_template directory (found inside the themes directory), and rename it to whatever you want your theme to be named. use single word names, or use hypens or underscores instead of spaces. For our example, we'll call our custom theme mytheme.
  • So, you should now have the following directory stucture: HelpSpot-Custom-Widget/themes/mytheme/
  • Inside of our newly created mytheme directory you will see another directory named images and a css stylesheet named style.css.
    • style.css is the CSS stylesheet used to override the "default" stylesheet and apply your theme's settings
    • The images directory is used to hold any images used specifically for your theme. NOTE: Unless you change the default behavior in the css stylesheet, then the script expects to find the following three images in the images directory of the theme you are using:
      • form-loading.gif - The image (usually an animated .gif) that shows while the form is being loaded
      • form-submitting.gif - The image (usually an animated .gif) that shows while the form is being submitted
      • close.gif - The image that's used for the "close" button on the popup
  • Edit your theme's style.css file to change the appearance of your form
    Using our example theme we created above, the file would be located at HelpSpot-Custom-Widget/themes/mytheme/style.css.
    Below are some of the basic elements you can edit that will change the appearance of your form. More advanced CSS users should be able to take what they learn here and customize their forms even further.
    • CSS Selector Description Notes
      #custom_widget_content This is the actual "lightbox" that pops up and that contains all of the content. Tthe border size and color are set here:
      (border: 10px solid #E0CF14;)
      #cw_form_container p#view_ticket This is the "box" of text that the user sees when they successfully submit a ticket. This element contains the user's AccessKey and the link to view their ticket.
      .cw_form.loading, #custom_widget_content.loading This describes the "loading" state of the form. It only applies whenever the form content is loading This is where you would define your image (animated gif?) to show whenever the form is loading.
      Note: This is where, by default, form-loading.gif is used
      .cw_form span#cw_status_text.loading The line of text that is shown when the submit button is clicked and the form is processing. It's a good idea to define a "loading" image (animated gif?) as well as the text color to use.
      Note: This is where, by default, form-submitting.gif is used
      a#custom_widget_close This is the "close" button that is used to close the form and lightbox If you are using the default style of close button then be sure to change the background color of this element to match the border color of #custom_widget_content
      Note: This is where, by default, close.gif is used

  • Make sure to upload your new theme directory (including all of its files and subdirectories) to the themes/ directory of wherever you uploaded your HelpSpot Custom Widget files.
  • Finally, to start using your new theme just follow the steps above on "How To Use Themes" -- replacing {theme_name} with the name of your newly created theme (probably "mytheme" if you've been following along the entire time)

Using, Customizing, and Creating Forms

This plugin allows you to create many different forms, and then dynamically set which form gets shown at run time. The main reason for this feature, besides allowing complete customization of the forms, is so that you can create different forms (ie: a form for support, one for suggestions, and one for comments) and then have the appropriate form show depending on what action the user takes.

Getting Started with Forms -- Things You Should Know!

A major change in the new version of this plugin is the way that forms are handled. To allow for even futher control and customization, forms are, for the most part, "self contained" now. Basically, each form (even any custom form you might create) is now responsible for the following:

  • Printing the HTML for the form itself
  • Setting up and handling all AJAX and jQuery/Javacript
    • Form validation rules
    • Formatting the form data to get it ready to submit
    • Submitting the form data to the HelpSpot API (by default this is done through the file ajax.php.)
  • Handling any messages that are returned from the HelpSpot API (or any other source, for that matter)
    • By default, this is done by using hidden <div> tags (see the default form.php to get a better understanding of this)

The first thing you should do is get more familiar with how the forms work. You can do this by examining the default form.php.

Forms and their Dependence on AJAX

Since the forms are now loaded in a "lightbox" instead of an iFrame, you can no longer use the form's default "submit" function to submit, validate, and process the form. The only feasible solution to this was to use AJAX to handle everything -- the submission, validation, processing of the form data, etc.

In the grand scheme of things, this isn't really a problem. I'll tell you why:

  • This gives us the ability to validate the form data before the form is ever submitted -- this not only provides a better user experience, but it also cuts down on the number of times that a page must be loaded (ie: this will also help to decrease server usage)
  • Speaking of validating data, our new AJAX dependency "problem" gave me the ability to implement the jQuery AJAX Validation library to make it even easier to validate your form fields
  • For those security-conscience users, you have the ability to use the ajax.php file (or whatever file you define) to perform server-side data validation
  • Are you worried about users with older browsers that don't support AJAX/Javacript? Don't! They won't even be able to see the widget in the first place!

Forms and the AJAX Validation Option

As mentioned above, the new version of this plugin has the jQuery AJAX Validation library included with it. While it is technically an option to use this library, it's highly recommended that you do! Mainly, because choosing to not use this option will create a ton of extra work for you. You see, not only does the validation feature validate the data, but it also takes care of displaying errors, submitting the form data to the API, handling the response, and much, much more... Basically, it does pretty much everything for you. If you choose to not use the AJAX Validation option then you will have to write the code to handle all of that yourself.

Using AJAX Validation on Your Forms

There are basically two ways of using the jQuery AJAX validation plugin within your forms: The easy way, and the "more technical" way. Let's start with the former:

  • AJAX Validation: The Easy Way:
    The easiest way to implement AJAX validation is to simply be aware of how you name your form fields when creating your forms. Just adding certain class names to your form fields will cause the AJAX validation to automatically take effect! Let's look at some examples:

    Say you have a field for the user's first name that you want to be required. All you have to do is add the class "required":
    <input type="text" name="firstname" class="required" value="" />

    Ok, that's cool enough, but what if you have a field for the user's email that needs to be required AND needs to be a valid email address? No Problem! Just add the classes "required" and "email":
    <input type="text" name="email" class="required email" value="" />

    You can learn more about all kinds of cool stuff like this at the AJAX Validation Library's author's page.

  • AJAX Validation: The "More Technical" Way:
    Because we actually include the AJAX Validation plugin just like you would on any other page or site, you have full access to its options, API, methods, etc. In other words, you have 100% complete control over the AJAX Validation plugin. Be forewarned though, manually configuring the plugin is rather intimidating until you've done it a few times.

    As much as I love the validation plugin, its main downfall is its [lack of] documentation. To be honest, I learned more about how to use the plugin by viewing the source code of the plugin's demo pages. To get a better understanding of how to set custom rules, messages, and more, I would recommend you do the same.

    If you already know what you're doing, or you want to give it a try anyways, you can add your custom validation rules to form.php (or whatever you've named your custom form) starting around line 167 (assuming that you're using the unedited code copied from form.php) -- either way, you will see a comment in the source code that tells you where to add your custom rules.

Creating and Using a Custom Form

To create a custom form it's recommended that you have experience with HTML, PHP, CSS, and a little jQuery knowledge wouldn't hurt either. Otherwise, you may find the required process a bit intimidating.

It's highly recommended that you start with a copy of the default form.php:

  • Start by creating a new file inside the Helpspot-Custom-Widget/ directory. For our example, we'll name our file custom_form.php.
    So, now we should have something like: Helpspot-Custom-Widget/custom_form.php
  • Next, copy the entire code from the original form.php and paste it into our newly created custom_form.php file.
  • Go ahead and save the changes made to custom_form.php.
    ProTip: Get in the habit of saving your work often!

At this point, it's probably best to read over the code, line-by-line, in custom_form.php (or whatever you named your file) to get an understanding of what everything does. Assuming that you're using the unedited code from the original form.php, then the code is pretty well commented, so hopefully that will help you (and most of the comments are in PHP format, so they won't be printed in the page's code if you decide to leave them). Once you've got a good understanding of how it all works, you can start making changes such as adding or removing form fields, and more.

Ready to start making your own forms? Before you start, here's a few things to keep in mind:

  • Please make sure to read (and fully understand) the section above titled Getting Started with Forms -- Things You Should Know.
  • If you're going to use the jQuery AJAX Validation library (I recommend you do), then consider doing it the easy way unless you need more control.
  • Remember to define your custom form when setting up the plugin (See the code below -- Line #7)

Telling the Plugin to Use Your New Form

			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       form: 'custom_form.php'
			   });
			   
			});	
			</script>

Custom AJAX Files for Your Forms

Each form uses an external file (ajax.php by default) to handle certain actions -- mainly to submit the form data to the HelpSpot API. If you based your custom form off the original form.php, then there is a chance that the default ajax.php will work for you.

If the default ajax.php doesn't work for you then you'll need to create your own. Unfortunately, this is not a "beginner" task. It will require PHP knowledge, as well as jQuery/AJAX knowledge.

If you do have to use a custom file for your AJAX functions be sure to include it when setting up the plugin:

			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       form: 'custom_form.php',
			       ajax: 'custom_ajax.php'
			   });
			   
			});	
			</script>

Forms, Forms, and More Forms!

Remember that you can have as many custom forms as you'd like -- Just repeat the process above whenever you want to create a new form. Just make sure that each new form you create has a unique name!

Creating Custom Tabs

A .psd file has been included if you would like to create your own custom tabs. Here's how to do it:

  • Locate the .psd template at HelpSpot-Custom-Widget/images/button-template.psd and open it in Photoshop
  • Once in Photoshop, select the text and change it to whatever you want. Keep in mind though, you only have about 95px of height to work with
  • Make sure that no background image is showing (ie: it has a transparent background)
  • Go to File > Save for Web & Devices (or hold down ctrl+alt+shift+s)
  • When the next screen pops up, choose PNG-24 and check Transparency:
  • Next, hit the Save button, which will popup another window where you can name your new image
  • It's recommended that you follow the existing naming convention, which is: button-{button_text}.png
    For example, if the text on your new tab is "FAQs" then you should name your image button-faqs.png (all lowercase)
  • Save the image somewhere to your computer, and then use an FTP program to upload it to HelpSpot-Custom-Widget/images (or wherever you uploaded the plugin files)
  • Edit widget.css and include the following code (this code uses the newly created "FAQs" tab from above. You may need to adjust accordingly)
    						#custom_widget_tab.faqs { background-image: url(images/button-faqs.png); }
  • Finally, you need to tell the plugin to use your new button:
			<script type="text/javascript">
			    $(function(){
			        $('html').HelpSpotWidget({
			            host: 'http://support.yourdomain.com/',
			            base: 'HelpSpot-Custom-Widget/',
	                    tab_type: 'faqs',
			        });
				   
    	    });	
			</script>
